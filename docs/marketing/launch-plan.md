# Launch Plan: Emotional Balance (Нейро-Психолог 24/7)

## 1. Launch Philosophy

### 1.1 Философия запуска

Emotional Balance — ИИ-платформа психологической поддержки 24/7 через Telegram WebApp на базе российских LLM (GigaChat/YandexGPT). Запуск строится на трёх фундаментальных принципах:

1. **Product-Led Growth через Telegram** — продукт сам является главным каналом привлечения. Нулевой барьер входа (Telegram-native), freemium-модель и реферальная программа через deep links обеспечивают органический рост. Пользователь может начать пользоваться продуктом за 30 секунд без установки приложений и регистрации.

2. **Wave-based Launch** — запуск проходит волнами, каждая из которых расширяет аудиторию и углубляет product-market fit:
   - Волна 1 (Alpha): внутреннее тестирование core-функций (Sprint 1-6)
   - Волна 2 (Closed Beta): 200-500 приглашённых пользователей (Sprint 7-13)
   - Волна 3 (Open Beta): расширение до 2000-5000 пользователей (Sprint 14-15)
   - Волна 4 (Public Launch): открытый доступ для всех (Sprint 16-17)
   - Волна 5 (Growth): масштабирование (Sprint 18+)

3. **Content-led Trust Building** — доверие к ИИ-психологу строится через экспертный контент задолго до запуска. Telegram-канал, PR-статьи и коллаборации с психологами формируют репутацию ещё до того, как пользователь впервые напишет боту.

## 3. Closed Beta (T-4w -> Launch)

### 3.1 Обзор Closed Beta

**Цель:** Валидировать продукт с реальными пользователями, собрать обратную связь, итерировать messaging, подготовить social proof для публичного запуска.

**Период:** 4 недели (соответствует Sprint 7-13 из Roadmap).

**Бюджет:** 200K руб.

**Целевая аудитория бета:** 200-500 пользователей, invitation-only.

#### Критерии отбора бета-тестеров

```yaml
Обязательные:
  - Telegram-аккаунт
```

### 3.5 Итерации по NPS

#### NPS-трекинг

| Неделя | Target NPS | Действия при NPS < 30 | Действия при NPS >= 40 |
|--------|-----------|----------------------|----------------------|
| Неделя 1 | >= 20 | Экстренный анализ детракторов, фокус на critical bugs | Собирать testimonials от промоутеров |
| Неделя 2 | >= 30 | Итерации UX на основе фидбека, улучшение онбординга | Активировать реферальные механики |
| Неделя 3 | >= 35 | A/B-тесты messaging, персонализация | Подготовить кейсы для PR |
| Неделя 4 | >= 40 | Последние критические фиксы | Go/No-Go решение для launch |

#### Принятие решения Go / No-Go

```yaml
Go (запуск):
  - NPS >= 40
  - D7 retention >= 30%
  - Critical bugs: 0
  - SOS-протоколы: протестированы, 0 false negatives на critical cases
  - Детектор кризисов: recall >= 95%
  - Нагрузочное тестирование: пройдено
  - 30+ testimonials собрано
  - Launch assets готовы

No-Go (задержка 1-2 недели):
  - NPS < 30
  - D7 retention < 20%
  - Critical bugs > 0
  - SOS/детектор кризисов: не прошли audit
  - Launch assets не готовы
```

## 9. Risk Mitigation

### 9.1 Технические риски запуска

| Риск | Вероятность | Влияние | Митигация | Ответственный |
|------|-------------|---------|-----------|---------------|
| Перегрузка серверов при launch spike | Средняя | Критическое | Нагрузочное тестирование на 10K concurrent, auto-scaling, CDN, graceful degradation | SRE / DevOps |
| LLM API downtime (GigaChat/YandexGPT) | Средняя | Высокое | Fallback между двумя LLM, кеширование типовых ответов, SOS-протоколы работают оффлайн | Dev / SRE |
| Telegram API rate limits | Низкая | Среднее | Queue management, exponential backoff, мониторинг rate limits | Dev |
| YooKassa сбой при пике оплат | Низкая | Высокое | Retry-механизм, уведомление пользователя, ручной разбор | Dev / Support |
| Потеря данных пользователей | Очень низкая | Критическое | Ежедневные бэкапы, PITR, репликация PostgreSQL | DevOps / SRE |

#### Технический план действий при перегрузке

```yaml
Уровень 1 (response time > 3 сек):
  - Алерт SRE
  - Отключение non-critical фич (статистика, инсайты)
  - Увеличение ресурсов (вертикальное масштабирование)

Уровень 2 (response time > 10 сек):
  - Алерт всей команде
  - Ограничение новых регистраций (waitlist)
  - Приоритизация SOS-запросов
  - Снижение маркетинговых активностей

Уровень 3 (сервис недоступен):
  - Incident response protocol
  - Статическая страница "Мы скоро вернёмся"
  - SOS-редирект на телефон доверия
  - Коммуникация в Telegram-канале
  - Post-mortem после восстановления
```

### 9.3 Продуктовые риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Низкий adoption (MAU < 3000 к месяцу 3) | Средняя | Высокое | A/B-тесты онбординга, усиление aha-момента, увеличение organic-каналов |
| Высокий churn (> 15% для платящих) | Средняя | Высокое | Engagement loops, персонализация push, winback-серии, exit surveys |
| Низкая конверсия Free-to-Paid (< 2%) | Средняя | Критическое | A/B-тесты paywall, триал-период, улучшение premium-фич |
| Плохое качество ИИ-диалогов | Средняя | Высокое | RAG-база КПТ, итерации промптов, экспертный аудит, feedback loops |
| Психологи не регистрируются на маркетплейсе | Средняя | Среднее | 0% комиссия на старте, прямой onboarding, вебинары для психологов |

### 9.4 Маркетинговые риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| CAC выше прогноза (> 300 руб) | Средняя | Высокое | Увеличить долю organic/referral, оптимизировать креативы, тестировать новые каналы |
| Стигма сильнее ожидаемого | Средняя | Среднее | Контент-стратегия нормализации, анонимность как USP, истории реальных людей |
| Telegram ограничивает WebApp | Низкая | Критическое | Начать разработку web-версии как backup |
| Ясно/Zigmund запускают ИИ-конкурента | Средняя | Высокое | Ускорить захват ниши, фокус на 5 уникальных фич, community lock-in |
| B2B цикл продаж дольше прогноза | Высокая | Среднее | Начать outbound на 2 месяца раньше, бесплатные пилоты, снизить friction |

### 9.5 Матрица реагирования

```
                    Высокое влияние
                         |
    [Серверная перегрузка]|  [Негативный PR]
    [Низкий adoption]    |  [Конкурент с ИИ]
                         |  [Низкая конверсия F2P]
    ─────────────────────┼──────────────────────
    Низкая вероятность   |  Высокая вероятность
                         |
    [Потеря данных]      |  [CAC > 300 руб]
    [TG ограничивает]    |  [B2B долгий цикл]
                         |  [Churn > 15%]
                         |
                    Низкое влияние
```

**Приоритет реагирования:**
1. Правый верхний квадрант: активный мониторинг + готовый план действий
2. Левый верхний квадрант: план действий + регулярная переоценка
3. Правый нижний квадрант: мониторинг + постепенная оптимизация
4. Левый нижний квадрант: принять риск + иметь fallback
