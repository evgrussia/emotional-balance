# Спецификация требований — Emotional Balance (Нейро-Психолог 24/7)

## 1. Введение

### 1.1 Назначение документа

Настоящий документ формализует полный набор требований к проекту **Emotional Balance** — ИИ-платформе психологической поддержки 24/7, функционирующей через Telegram WebApp на базе российских LLM (GigaChat, YandexGPT). Документ структурирован в соответствии с практиками BABOK (Business Analysis Body of Knowledge) и охватывает бизнес-требования, требования стейкхолдеров, функциональные и нефункциональные решения, а также переходные требования.

### 1.2 Scope (Границы)

| Параметр | Значение |
|----------|----------|
| Продукт | ИИ-платформа психологической поддержки 24/7 |
| Платформа | Telegram WebApp (MVP) |
| Модули | 18 FR (14 P0 + 4 P1), 47 User Stories |
| Персоны | 6 целевых персон (4 B2C + 1 B2B + 1 Админ) |
| Стейкхолдеры | 15 групп (S01-S15) |
| Монетизация | Freemium: Free / Standard 990 руб/мес / Premium 2990 руб/мес + B2B 1500 руб/сотр + Marketplace 15% |
| Финансирование | Seed-раунд 15 млн руб на 6 месяцев |
| Tech stack | Django + React + PostgreSQL + GigaChat/YandexGPT + LangChain |
| Compliance | ФЗ-152, хранение данных в РФ |

**В scope:**
- 24/7 текстовый и голосовой чат с ИИ-психологом
- Дневник эмоций с ИИ-анализом паттернов
- КПТ-упражнения адаптированные под пользователя
- Детектор кризисных состояний с эскалацией (Safety-Critical)
- SOS-протоколы быстрой помощи
- Маркетплейс верифицированных психологов
- Терапевтический мост ИИ-Психолог
- Медитации и дыхательные практики
- Интеграция с носимыми устройствами (Apple Watch, Mi Band)
- Подписки и платежи через YooKassa
- B2B корпоративное управление
- Мини-курсы самообразования
- Реферальная программа
- Терапевтическая геймификация
- Система лимитов и контроля расходов

**Вне scope (MVP):**
- Нативные мобильные приложения (iOS/Android)
- Веб-версия вне Telegram
- Работа с несовершеннолетними
- Международная экспансия
- Психиатрическая помощь и назначение медикаментов

### 1.3 Методология

Требования классифицированы по четырём уровням:

| Уровень | Описание | Кодировка |
|---------|----------|-----------|
| Business Requirements | Стратегические бизнес-цели проекта | BR-001...BR-010 |
| Stakeholder Requirements | Потребности и ожидания стейкхолдеров | SR-001...SR-015 |

Приоритизация: MoSCoW (Must/Should/Could/Won't).

Traceability: каждое требование нижнего уровня трассируется к вышестоящему.

## 2. Business Requirements (BR)

Бизнес-требования определяют стратегические цели проекта, которыми руководствуются все нижестоящие требования.

| ID | Требование | Описание | Метрика успеха | Приоритет |
|----|-----------|----------|---------------|-----------|
| BR-001 | Обеспечить 24/7 доступ к психологической поддержке через Telegram | Платформа должна предоставлять круглосуточный доступ к ИИ-психологу через Telegram WebApp для всех категорий пользователей | Uptime >= 99.5%; Время первого ответа < 2 сек; Доступность 24/7/365 | Must |
| BR-002 | Достичь BEP к месяцу 5-6 при ~1850 платящих пользователях | Финансовая модель должна обеспечить операционную безубыточность в рамках Seed-раунда (15M руб / 6 мес) | BEP <= месяц 6; Платящие >= 1850; MRR >= 2.5M руб | Must |
| BR-003 | Обеспечить compliance ФЗ-152 | Все персональные данные пользователей должны обрабатываться и храниться в соответствии с требованиями ФЗ-152 на территории РФ | 0 нарушений ФЗ-152; 100% данных в РФ; Регистрация как оператор ПДн | Must |
| BR-004 | Обеспечить безопасность пользователей (кризис, SOS) | Система должна автоматически детектировать кризисные состояния и обеспечивать немедленную эскалацию на специалистов и горячую линию | Crisis detection accuracy > 95%; False negative rate < 1%; Response time < 60 сек | Must |
| BR-005 | Создать гибрид ИИ + маркетплейс (уникальная позиция) | Платформа должна сочетать ИИ-психолога для первичной поддержки с маркетплейсом живых психологов для углублённой работы | >= 20 психологов на запуске; Конверсия ИИ-чат -> маркетплейс > 5%; Средний рейтинг психологов > 4.5 | Must |
| BR-006 | Обеспечить B2B wellness-канал | Платформа должна предоставлять корпоративным клиентам wellness-программу с агрегированной аналитикой без раскрытия персональных данных | >= 5 B2B клиентов к месяцу 6; B2B ARPU >= 75K руб/мес; CSAT B2B > 4.0 | Must |
| BR-007 | Достичь 50K MAU за 12 месяцев | Обеспечить рост пользовательской базы до 50 000 MAU через органические и платные каналы привлечения | MAU = 50K к месяцу 12; Retention D30 > 25%; Sessions/week > 3 | Must |
| BR-008 | Обеспечить scientific-based КПТ-подход | Все терапевтические интервенции должны основываться на научно обоснованных КПТ-протоколах, верифицированных клиническими психологами | PHQ-9/GAD-7 improvement > 20% за 8 нед; 100% протоколов верифицировано КПТ-экспертами; CSAT > 4.2 | Must |
| BR-009 | Минимизировать барьер входа (Telegram-first) | Регистрация и доступ должны осуществляться в один клик через Telegram без дополнительных установок | Time-to-first-value < 2 мин; Регистрация в 1 клик; 0 дополнительных установок | Should |
| BR-010 | Обеспечить масштабируемость (500K MAU к Году 3) | Архитектура системы должна поддерживать горизонтальное масштабирование до 500K MAU без переписывания ядра | MAU capacity до 500K; Горизонтальное масштабирование; Latency < 3 сек при пиковой нагрузке | Should |
