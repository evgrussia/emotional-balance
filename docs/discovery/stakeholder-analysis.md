# Анализ стейкхолдеров — Emotional Balance

## 1. Введение и цель документа

### 1.1 Назначение

Настоящий документ определяет всех заинтересованных сторон (стейкхолдеров) проекта **Emotional Balance** — ИИ-платформы психологической поддержки 24/7, функционирующей через Telegram WebApp на базе российских LLM (GigaChat, YandexGPT). Анализ охватывает идентификацию, классификацию, оценку влияния и вовлечённости каждого стейкхолдера, а также стратегию взаимодействия с ними.

### 1.2 Цели анализа

- Идентифицировать все группы стейкхолдеров проекта
- Оценить уровень их влияния и заинтересованности
- Определить стратегию коммуникации и вовлечения для каждой группы
- Минимизировать риски, связанные с неудовлетворёнными ожиданиями стейкхолдеров
- Построить RACI-матрицу для ключевых решений проекта

### 1.3 Контекст проекта

| Параметр | Значение |
|----------|----------|
| Продукт | ИИ-платформа психологической поддержки 24/7 |
| Платформа | Telegram WebApp |
| Модули | 18 FR (14 P0 + 4 P1), 47 User Stories |
| Персоны | 6 целевых персон |
| Монетизация | Freemium: Free / Standard 990 руб/мес / Premium 2990 руб/мес + B2B 1500 руб/сотр + Marketplace 15% |
| Финансирование | Seed-раунд 15 млн руб на 6 месяцев |
| Tech stack | Django + React + PostgreSQL + GigaChat/YandexGPT + LangChain |

### 2.2 Детализация по группам

#### A. Конечные пользователи (S01-S04)

**P1 Аня (28, маркетолог, тревожная миллениалка)**
- Сегмент: Молодые профессионалы с тревожностью
- Частота использования: ежедневно
- Готовность платить: Standard (990 руб/мес) после free-периода
- Критичные модули: ИИ-чат, дневник настроения, техники КПТ
- Точка входа: Instagram / Telegram каналы о ментальном здоровье

**P2 Дмитрий (35, руководитель, выгорающий)**
- Сегмент: Менеджеры среднего/высшего звена с профессиональным выгоранием
- Частота использования: 3-5 раз в неделю
- Готовность платить: Premium (2990 руб/мес), ценит эксклюзивность
- Критичные модули: ИИ-чат, маркетплейс психологов, аналитика прогресса
- Точка входа: LinkedIn / профессиональные сообщества

**P3 Марина (40, HR-директор, B2B)**
- Сегмент: Корпоративные клиенты (50-500 сотрудников)
- Частота использования: дашборд — еженедельно
- Готовность платить: B2B тариф 1500 руб/сотрудник в месяц
- Критичные модули: B2B дашборд, агрегированная аналитика, массовый onboarding
- Точка входа: HR-конференции, прямые продажи

**P4 Олег (32, инженер, житель региона)**
- Сегмент: Жители регионов с ограниченным доступом к психологам
- Частота использования: 2-3 раза в неделю
- Готовность платить: Free или Standard при доказанной ценности
- Критичные модули: ИИ-чат, самопомощь, образовательный контент
- Точка входа: Поиск / тематические Telegram-каналы

#### B. Поставщики услуг (S05, S07)

**P5 Елена (38, психолог) и пул психологов маркетплейса**
- Количество на запуске: 20-50 верифицированных специалистов
- Модель дохода: комиссия платформы 15% от стоимости сессии
- Критичные модули: кабинет психолога, календарь, оплата, отзывы
- Требования: верификация дипломов, страхование ответственности, модерация

#### C. Внутренние стейкхолдеры (S06, S11, S12)

**Команда разработки**
- Состав (MVP): 2 backend, 1 frontend, 1 ML/AI, 1 DevOps, 1 QA
- Tech stack: Django + React + PostgreSQL + LangChain + GigaChat/YandexGPT
- Методология: Agile/Scrum, 2-недельные спринты
- Мотивация: интересный стек, социальный импакт, опционы

**Контент-специалисты КПТ**
- Роль: разработка терапевтических сценариев, промптов, упражнений
- Требования: образование в клинической психологии, знание КПТ
- Формат: частичная занятость или консалтинг

#### D. Внешние партнёры (S13, S14, S15)

**Горячая линия кризисной помощи (8-800-2000-122)**
- Тип интеграции: мгновенная кнопка перенаправления при детекции кризиса
- Протокол: автоматическая эскалация при обнаружении суицидальных маркеров
- SLA: отображение контакта менее 1 секунды после детекции

**Провайдеры LLM**
- GigaChat (Сбер): основная модель для диалогов
- YandexGPT: резервная модель, A/B-тесты
- Модель оплаты: pay-per-token
- Критично: SLA 99.5%, latency < 3 сек

**YooKassa**
- Тип интеграции: подписки, разовые платежи, выплаты психологам
- Комиссия: 3.5% от транзакции
- Требования: PCI DSS, 54-ФЗ (онлайн-кассы)

### Пояснения к ключевым решениям

**D01 — Выбор LLM-провайдера:** Architect оценивает latency, стоимость токенов, качество русскоязычных ответов. Финальное решение за CEO с учётом бюджета Seed-раунда.

**D02 — Дизайн кризисного детектора:** Критически важное решение. Привлекаются клинические психологи (S12) для определения маркеров, горячая линия (S13) для протокола эскалации. Product Manager координирует.

**D03 — Ценообразование:** Основано на unit-экономике (docs/discovery/unit-economics.md). Консультации с инвесторами по целевым метрикам и с B2B клиентами по готовности платить.

**D04 — Соответствие ФЗ-152:** Architect обеспечивает техническую реализацию: хранение ПДн в РФ, шифрование, право на удаление. Юридическая консультация обязательна.

**D05 — Onboarding психологов:** Верификация дипломов, проверка лицензий, модерация профилей. Психологи консультируются по процессу.

**D08 — Решение о запуске:** Комплексное решение на основе Quality Gates: все P0-модули готовы, тесты зелёные, compliance подтверждён, мониторинг настроен.

**D10 — Аудит безопасности:** Включает penetration testing, проверку хранения ПДн, аудит промптов на jailbreak, верификацию шифрования.

## 6. Stakeholder Engagement Strategy (Стратегия вовлечения)

### 6.1 Конечные пользователи (S01-S04)

| Параметр | Значение |
|----------|----------|
| **Стратегия** | Community-driven development: вовлечение через обратную связь и co-creation |
| **Тактики** | Beta-программа (100-500 пользователей), NPS-опросы ежемесячно, in-app feedback widget, Telegram-сообщество, интервью с power users |
| **Уровень вовлечения** | Информирование + консультирование |
| **KPI вовлечения** | NPS > 50, Response Rate на опросы > 15%, Beta retention > 60%, Среднее время ответа на feedback < 24ч |
| **Риски** | Низкая активность в beta, негативные отзывы, churn после free-периода |
| **Митигация** | Gamification в beta, быстрый response loop, плавный onboarding в платные тарифы |

### 6.2 Психологи маркетплейса (S05, S07)

| Параметр | Значение |
|----------|----------|
| **Стратегия** | Партнёрство и взаимный рост: психологи — амбассадоры платформы |
| **Тактики** | Персональный onboarding, обучающие вебинары, реферальная программа, закрытый Telegram-чат, совместное создание контента |
| **Уровень вовлечения** | Активное партнёрство |
| **KPI вовлечения** | Время от регистрации до первого клиента < 14 дней, Средний рейтинг психологов > 4.5, Retention психологов на платформе > 80% за 6 мес, Количество сессий на психолога > 10/мес |
| **Риски** | Недостаточный поток клиентов, недовольство комиссией, репутационные инциденты |

### 7.3 Планы реагирования на топ-3 риска

**R01 — Утечка персональных данных**
```
Триггер: Обнаружение несанкционированного доступа к данным
Немедленно:
  1. Изолировать затронутые системы
  2. Уведомить CEO и DPO
  3. Оценить масштаб утечки
В течение 24 часов:
  4. Уведомить Роскомнадзор (если подтверждена утечка ПДн)
  5. Уведомить пострадавших пользователей
  6. Начать forensic-исследование
Долгосрочно:
  7. Устранить причину
  8. Усилить контроли
  9. Опубликовать post-mortem
```

**R02 — Некорректная работа кризисного детектора**
```
Триггер: Пропущенный кризисный случай или массовые ложные срабатывания
Немедленно:
  1. Снизить порог детекции (лучше false positive, чем false negative)
  2. Включить human-in-the-loop для всех подозрительных случаев
  3. Уведомить команду КПТ-контента
В течение 48 часов:
  4. Провести анализ причин
  5. Обновить модель детекции
  6. Протестировать на историческихданных
Долгосрочно:
  7. Мультимодельная валидация (LLM + rule-based + ML classifier)
  8. Регулярная калибровка (ежемесячно)
```

**R05 — Негативный PR из-за качества ИИ-ответов**
```
Триггер: Публичная жалоба на вредный/некорректный ответ ИИ
Немедленно:
  1. Удалить/исправить проблемный контент
  2. Связаться с пользователем
  3. Оценить потенциальный ущерб
В течение 24 часов:
  4. Добавить guardrail для предотвращения повторения
  5. Подготовить публичное заявление (если необходимо)
Долгосрочно:
  6. Усилить систему guardrails
  7. Расширить тестовые сценарии
  8. Внедрить red team тестирование
```
